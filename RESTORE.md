# TFC Backup & Restore Runbook

Static site — no database. Backups cover HTML, CSS, JS, and film pages.

## What is backed up

| Source | Schedule | Retention |
|--------|----------|-----------|
| GitHub (`origin`) | On every `git push` | Full git history |
| GitHub Actions artifact | Daily 02:00 UTC | 30 days |
| Local `backup.ps1` / `backup.sh` | Manual or scheduled | 14 daily folders |

Critical checksums are listed in `backup-manifest.json` (generated by `scripts/build-backup-manifest.js`).

## Restore from Git (recommended)

```bash
git clone https://github.com/ktiwari54/TFC.git
cd TFC
python serve.py
# Open http://localhost:8080/ and verify key pages
```

Redeploy production: push to `main` (Vercel auto-deploy) or roll back to a prior deployment in the Vercel dashboard.

### Single-file recovery

```bash
git log --oneline -- path/to/file
git checkout <commit-hash> -- path/to/file
```

## Restore from local backup

1. Locate `backups/YYYY-MM-DD/` or `backups/tfc-backup-YYYY-MM-DD.zip`
2. Extract/copy files over the project (or into a fresh folder)
3. Verify manifest:

```bash
node scripts/build-backup-manifest.js /tmp/verify-manifest.json
# Compare sha256 values with the manifest inside the backup folder
```

4. Start the site: `python serve.py`
5. Smoke-test: homepage, `/films.html`, `/contact.html`, `/films-search.html?q=priya`

## Restore from GitHub Actions artifact

1. GitHub → **Actions** → **Daily Backup** → latest run
2. Download artifact `tfc-daily-backup-*`
3. Extract `tfc-backup-YYYYMMDD.tar.gz` into a clean directory
4. Follow steps 3–5 above

## Enable daily backups on Windows (Task Scheduler)

Run once in PowerShell **as Administrator**, or use `scripts/register-daily-backup-task.ps1`:

```powershell
$action = New-ScheduledTaskAction -Execute 'powershell.exe' `
  -Argument '-NoProfile -ExecutionPolicy Bypass -File "C:\Users\PC\Desktop\TFC\backup.ps1"'
$trigger = New-ScheduledTaskTrigger -Daily -At 2:00AM
Register-ScheduledTask -TaskName 'TFC-Daily-Backup' -Action $action -Trigger $trigger `
  -Description 'Daily backup of TFC static site files'
```

Adjust the path if the project moves.

## Enable daily backups on Linux/macOS (cron)

```cron
0 2 * * * cd /path/to/TFC && ./backup.sh >> backups/backup.log 2>&1
```

## Database backup

Not applicable. TFC has no server-side database. Contact enquiries use `mailto:` and are not stored in this repo.

If a database is added later, extend `.github/workflows/daily-backup.yml` and `backup.ps1` with your dump command (e.g. `pg_dump`).

## Verify backup health

```bash
node test-backup-recovery.js
```

Expected: daily-backup and restore-procedure checks pass; database-backup is skipped (static site).
